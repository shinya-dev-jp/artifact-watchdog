# Integration Examples

This directory contains copyable templates for common scheduler and artifact-checking setups.

| Directory | Scenario | Files to copy |
|---|---|---|
| `github-actions` | Scheduled GitHub workflow with a failing CI gate. | `artifact-watchdog.yml` and `artifact-watchdog.toml` |
| `cron` | Daily cron job on a local machine or server. | `artifact-watchdog-cron.sh` and `artifact-watchdog.toml` |
| `systemd` | Linux timer managed by systemd. | `artifact-watchdog.service`, `artifact-watchdog.timer`, and `artifact-watchdog.toml` |
| `local-agent-report` | Daily report generated by an agent or automation script. | `run-agent-report-check.sh` and `artifact-watchdog.toml` |
| `release-notes` | Release notes that must exist before a release cut. | `run-release-note-check.sh` and `artifact-watchdog.toml` |

See [`docs/INTEGRATION_TEMPLATES.md`](../../docs/INTEGRATION_TEMPLATES.md) for selection guidance and the adaptation checklist.
